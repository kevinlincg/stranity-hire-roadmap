# Debugging Guide

## Goal

Learn how to debug issues systematically.

---

## Basic Workflow

1. Reproduce the issue
2. Read logs
3. Trace request flow
4. Narrow down the problem
5. Verify the fix

---

## Backend Debugging

Check:

- request payload
- database query
- middleware flow
- error logs

---

## Frontend Debugging

Check:

- network requests
- console logs
- component state
- rendering flow

---

## Common Mistakes

- guessing without verifying
- changing too many things at once
- ignoring logs

---

## Important Mindset

Good engineers do not panic during debugging.

They isolate problems step by step.
