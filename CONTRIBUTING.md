# Contributing Guide

## Branch Naming

請使用：

- feature/xxx
- fix/xxx
- refactor/xxx

例如：

- feature/login-api
- fix/strategy-pagination

---

# Commit Message

請避免：

- update
- fix bug
- test

請使用：

- feat: add login api
- fix: resolve jwt middleware issue
- refactor: simplify strategy service

---

# Pull Request

PR 應包含：

- 修改內容
- 修改原因
- 測試方式
- screenshot（frontend）

---

# Code Review

請注意：

- review 是合作，不是批評
- maintainability 比 clever code 更重要
- naming clarity 很重要

---

# Engineering Mindset

比起：

「先能動」

更重要的是：

- maintainable
- readable
- scalable
